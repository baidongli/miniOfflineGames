# iOS Nearby Connections Bridge

Swift framework wrapping Google's official `NearbyConnections` CocoaPod,
exposing a flat C ABI to Unity via `[DllImport("__Internal")]`.

## Files

- `NearbyBridge/NearbyBridge.swift` — main bridge class + `@_cdecl` exports.
- `NearbyBridge/NearbyBridge_UnityShim.m` — calls `UnitySendMessage` (resolved at link time from Unity's host binary).
- `NearbyBridge/Info.plist` — framework metadata.
- `Podfile` — declares the `NearbyConnections` pod dependency.

## Build

The Xcode project file is **not** committed (generated by `xcodegen` or hand-crafted in Xcode). To build:

1. Create a new "Framework" Xcode project named `NearbyBridge` here.
2. Add `NearbyBridge.swift`, `NearbyBridge_UnityShim.m` to the target.
3. From this folder: `pod install`.
4. Open `NearbyBridge.xcworkspace`.
5. Build: Product → Archive (Release, iOS device).
6. Copy the resulting `.framework` (and required pod frameworks) into
   `unity/Assets/Plugins/iOS/`.

Alternatively, integrate the source files directly into the generated Unity
Xcode project (post-build script) — slightly faster iteration but less clean.

## Wire conventions

Unity GameObject receiving callbacks must be named `_NearbyTransportReceiver`.
Callback payload formats mirror the Android bridge — see
`unity/Assets/Networking/Transport/NearbyConnectionsTransport.cs`.

## Required Info.plist additions in Unity build

These go in the **Unity** project's iOS player settings → "other settings" →
custom plist entries (or via a post-build script):

- `NSLocalNetworkUsageDescription` — "Find nearby players to play with."
- `NSBluetoothAlwaysUsageDescription` — "Connect to nearby devices for multiplayer."
- `NSBonjourServices` array — entries like `_<serviceId>._tcp`, `_<serviceId>._udp`.
