# 2048 (Number Merge)

Single-player puzzle with multiplayer score-race variant.

## Rules

- 4 x 4 grid, starts with 2 random tiles (both "2" usually; rarely "4").
- Swipe up/down/left/right: every tile slides as far as possible in
  that direction. Adjacent tiles with the **same value** merge into a
  tile of double the value. Each tile can merge at most once per swipe.
- After a swipe that moved or merged something, a new "2" (90%) or "4"
  (10%) spawns at a random empty cell.
- Reaching a 2048 tile = victory popup (game continues if you want).
- Game over when no swipe direction would move or merge anything.

## Storage trick

Cells store the **exponent** instead of the value: 0 = empty, 1 = "2",
2 = "4", 3 = "8", ..., 11 = "2048". One byte per cell, all the way up
to "65536" (exponent 16). Display layer converts via `1 << exponent`.

## Controls

- Swipe in 4 cardinal directions, or press arrow keys (Editor only).

## Multiplayer (score race)

Each peer runs an **identically-seeded** game so the initial layout and
the entire tile-spawn sequence is identical across all peers - the only
variable is who plays best. Players broadcast:

- `NMProgressMessage` after each swipe (score, max exponent, swipe count)
- `NMReachedGoalMessage` the first time they hit 2048
- `NMDiedOutMessage` on game-over

Winning condition: first to a target exponent (typically 2048), or
highest score after a fixed swipe budget / timer (TBD).

## AI

`SimpleNumberMergeAI` is a 1-ply lookahead with a classic 2048 heuristic:
- More empty cells = better.
- Max tile pinned in a corner = +500.
- Monotonic bottom row = small bonus per descending pair.

Strong AIs need expectimax 3-5 plies; that's room for future work.
